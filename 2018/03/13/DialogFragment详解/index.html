链接为 v4 支持库，因为在 Android P 中 非支持包下的 DialogFragment 将被废弃。

1. DialogFragment 取代 Dialog 的必要性

在官方文档中，官方推荐我们使用 DialogFragment 来代替 Dialog，主要是 AlertDialog ，因为 ProgressDialog 在 API 26 中被废弃，官方希望我们使用 ProgressBar 来代替它。

为什么要使用 DialogFragment 来代替 Dialog 来实现对话框，从文档中可以看到以下使用 DialogFragment 几个好处：

1.DialogFragment 继承自 Fragment，有着一致的生命周期，可以方便 Activity 对其进行生命周期管理（屏幕旋转、返回键操作等）；

2.DialogFragment 允许开发者把 Dialog 作为内嵌的组件进行重用，类似 Fragment（可以在大屏幕和小屏幕显示出不同的效果）。onCreateDialog() 即利用 AlertDialog 或者 Dialog 创建对话框。

2.如何创建一个 DialogFragment

有两种方式可以创建一个 DialogFragment：

  • 重写 onCreateDialog() 方法，内嵌 AlertDialog 对象，返回一个 Dialog 实例

  • 重写 onCreateView() 方法，返回一个自定义对话框的视图 View

3.如何使用

使用方式非常简单，只需创建 DialogFragment 的实例，然后调用 show() 将其显示在 Activity 当中。

还有另外一个方式，在Activity 布局中设置 fragment ，然后通过正常的 Fragment 的调用方式获取，然后调用。

5.其它问题

在自定义 Dialog 样式的使用过程中，经常使用getDialog().requestWindowFeature(Window.EATURE_NO_TITLE)来隐藏对话框中的标题栏，但是这个操作，会引起自定义样式布局的根节点的固定的 layout_width属性值不起作用，会自动按照layout_width = wrap_content来进行显示。
暂时想到解决的方法就是：在子控件布局中指定准确的布局的长宽，或者在 onActivityCreated() 生命周期方法中修改想要的尺寸。
