Annotation简介

注解是Java 5的新特性。它是一种元数据，是一种代码级别的说明。

注解的层次等级和类、接口、枚举同属一个层次。

注解的作用

  • 编写文档：生成doc格式文档
  • 代码分析：通过反射来分析代码
  • 编译检测：Override

注解的作用范围：包、类、字段、方法、局部变量以及方法参数。

JDK提供的三种注解

@Override

方法的重写。接口、抽象类方法的实现、类方法的重写。

@Deprecated

修饰一个方法，表示该方法可用，但是过时的方法，不提倡使用。

@SuppressWarnings

忽略编译器给出的编译的警告。
该注解拥有个参数，一般写作:@SuppressWarnings(参数)。6个参数分别如下：

  • deprecation 忽略过时
  • rawtypes 忽略类型安全
  • unused 忽略不使用
  • unchecked 忽略安全检查
  • null 忽略空指针
  • all 忽略所有编译警告

    @注解名(属性1 = 属性值1， 属性2 = 属性值2，…)
    需要注意的是，当属性名为value时，且注解只有一个属性，那可以缩写成@注解名(属性值)。同时，一个对象，注解只能使用一次，不能重复。

元注解

元注解就是用于修饰自定义注解的注解，JDK提供了四种元注解

@Retention

用于修饰自定义注解的生命周期，只有一个属性，有三种属性值，也称三种策略：

  • RetentionPolicy.SOURCE 被修饰的注解只存在源码中
  • RetentionPolicy.CLASS 被修饰的注解只存在于源码和字节码文件中
  • RetentionPolicy.RUNTIME 被修饰的注解存在源码、字节码、内存(运行时)中

@Target

用于确定自定义注解的作用的位置，只有一个属性，由于被修饰的位置有八种，因此也有八种属性值：

  • ElementType.TYPE 类和接口
  • ElementType.METHOD 方法
  • ElementType.FIELD 字段
  • ElementType.PARAMETER 方法参数
  • ElementType.CONSTRUCTOR 构造
  • ElementType.LOCAL_VARIABLE 修饰本地变量或catch语句
  • ElementType.ANNOTATION_TYPE 修饰注解类型
  • ElementType.PACKAGE 修饰包

@Documented

使用javaDoc生成api文档时，是否包含该注解。无属性。

@Inherited

父类使用修饰的注解，子类是否继承。无参数。

☆自定义注解解析AnnotatedElement

用于获取注解上设置的数据，用过反射来操作。以下是几个反射的方法。

  • boolean isAnnotationPresent(Class annnotationClass); 判断当前对象是否有该注解。
  • T getAnnotation(Class<T> annotationClass); 获取当前对象上指定的注解。
  • Annotation[] getAnnotation(); 获取当前对象及其父类上继承的所有注解。
  • Annotation[] getDeclaredAnnotations();获取当前对象上所有的注解，不包括继承的注解。
