开始前的准备工作：正常工作的Mac，以及其上的 Git 版本控制工具、HomeBrew 工具。

部署博客系统到 Github

现在可以本地浏览个人博客了，下一步将个人博客系统部署到 Github 中

1.建立博客系统仓库

在 Github 个人账号下新建一个仓库，命名为：**[你的用户名].github.io** ， 例如：tommyfen.github.io,

建立完毕后，复制项目的 ssh 仓库地址(https 仓库地址也一样使用)。

2.本地博客系统关联Github仓库

在本地博客系统文件夹 blog 目录下，找到： _config.yml 博客系统配置文件，修改 deploy 相关内容并保存：

3.安装 git 相关 hexo 组件

在本地博客系统当前文件根目录下，打开终端：

4.部署博客到Github

先生成静态资源，然后将文件部署到Github当中：

5.在线访问个人博客

在 Github 上仓库中，选择 Setting，找到 Github Pages 选项，配置 Source 选项，默认是 None，我们需要将其修改 master branch（上述_config.yml配置中branch为master，建议开始时不熟悉保持master作为选项），配置完成之后，若想要访问网站是通过 https 加密保护，将 Source 选项下的 Enforce HTTPS 勾选上。

然后在浏览器输入：[用户名].github.io 就能访问 例如：tommyfen.github.io

至此，个人博客部署搭建完毕，可以通过 xxx.github.io 的方式进行访问

绑定个人域名

绑定域名是个挺折腾人的事情，在配置文件配置完毕之后，还需要在域名购买商进行域名解析。 同时购买域名需要实名，在国内作为外链使用时，还需要进行户籍所在地的 ICP 备案。

1.首先，需要在本地博客系统文件夹blog内的 _config.yml 文件进行配置。 找到 #url 模块， 添加： url: [个人域名]

2.然后本地博客系统文件文件夹blog的 source 目录 新建一个文件，命名为： CNAME。 并在文件内写入个人域名，例如：tommyfen.cn

3.然后通过域名售卖商（例如：阿里云）进行解析，如下图所示：

4.在 Github 仓库上绑定你的个人域名:
在Github仓库中选进入 Setting 配置，然后找到 Github Pages，其中一项叫做 Custom domain，在其上输入你的个人域名，例如：tommyfen.cn 。然后点击 Save 保存。

5.最后在本地博客系统文件夹blog目录下打开终端：
